# Kotlin 기초 개념

oop 언어 객체지향 프로그래밍 

5가지 기본 개념은 변수와 타입, 흐름 제어, 함수, 컬렉션, 상속을 포함한 클래스와 객체가 있습니다

변수는 저장 공간에서 위치입니다

변수를 만들면, 컴퓨터 어딘가에 저장되는 거죠

저장 공간을 나타내기 위해 고유의 이름을 정해줘야 합니다

identifier(id)라는 이름을 꼭 넣어야 하는 거죠

이름으로 데이터에 접근할 수 있어요

이제 'fun main'을 살펴볼게요 메인 함수를 말하는 거죠

간단히 시작 점입니다 코틀린 앱 또는 프로그램을 만들 때, 프로그램의 시작점이 필요한데

그것이 바로 메인 함수입니다 즉, 코틀린 프로그램을 만들 때 항상 보게 될 부분이에요

이제 흐름 제어를 볼게요

흐름 제어는 조건이 맞는 경우에만 코드를 실행해주고

또 조건이 맞는 한 코드를 반복 실행하게 해줍니다

특정 코드를 여러 번 반복 실행할 때가 많아 매우 유용합니다

다음, 함수를 볼게요 함수는 코드를 나눠주고 필요시 코드 블록을 실행하게 해줍니다

좋은 점은 코드를 쉽게 나누게 해주고 정리해준다는 겁니다

이러면 코드를 재사용하는 데 도움이 많이 되죠 즉, 많이 입력을 안 해도 된다는 거예요

물론, 괄호들이 뭔지, 함수 이름을 어떻게 지을지,

함수 호출 방법, 호출한다는 건 뭔지 등을 모두 배울거예요

이건 함수의 작은 예시죠 그럼 다음으로 컬렉션이 있습니다

컬렉션은 많은 요소를 한 군데에 저장하게 해주고 흐름 제어의 도움을 받아 여러 요소를 반복 실행하게

해줍니다 즉, 한 곳에 여러 값을 저장하고 싶은 경우 예를 들어, 한 곳에 학생 이름 모두 저장하고 싶으면

컬렉션을 사용하는 겁니다

다음으로 클래스가 있습니다 클래스는 직접 데이터 타입을 만들게 해주고, 데이터 멤버와 메서드를 한 곳에 있게 해줍니다

메서드는 알아보기 쉽고 유지 가능한 코드를 더 만들게 해줍니다

프로젝트를 팀으로 작업할 때 정말 유용한 거죠

프로젝트가 커질수록, 더 복잡해지기 때문에 우리가 모든 걸 이해할 수 있게 정리해 놓는 것이 아주 중요해져요

그러기 위해 클래스가 필요한 거죠

객체 지향 프로그래밍에서는 모든 것이 어느 정도 클래스로 되어 있는 게 중요합니다
